# Landing Examples for Marketing Site

> Ця папка **не є частиною основного застосунку**. Це окремі React‑компоненти, які можна вставляти у лендінг CBT Self‑Supervision / Bravery Academy.

## Структура

- `LandingExampleCard.tsx` – спільна картка прикладу (фон, бордери, бейджі).
- `StudentsExamples.tsx` – 5 прикладів для **студентів / ординаторів / інтернів**.
- `ClinicsExamples.tsx` – 5 прикладів для **клінік / центрів / освітніх інститутів**.
- `PrivatePracticeExamples.tsx` – 5 прикладів для **приватних практикуючих психологів**.
- `CompaniesExamples.tsx` – 5 прикладів для **компаній / HR / корпоративних програм**.

Усі компоненти:
- написані на **React (TSX)**;
- не мають імпортів з `src/` основного застосунку;
- не викликають API, не працюють з реальними даними;
- містять **двомовні тексти (uk/en)**;
- мають **просту інтерактивність** (клік, тумблери, зміна стану) для демонстрації живого інтерфейсу.

## Як використати на лендінгу

1. **Скопіювати файли**
   - або перенести всю папку `landing-examples/` у репозиторій лендінгу;
   - або взяти тільки потрібний файл, наприклад `StudentsExamples.tsx`.

2. **Підключити компонент**

```tsx
// приклад для Next.js / React SPA
import { StudentsExamplesShowcase } from './landing-examples/StudentsExamples';

export function LandingSection() {
  return (
    <section className="bg-slate-950 py-16">
      <div className="container mx-auto px-4">
        <StudentsExamplesShowcase />
      </div>
    </section>
  );
}
```

3. **Стилі**
- Компоненти використовують прості utility‑класи у стилі Tailwind (`bg-slate-950`, `rounded-xl` тощо).
- Якщо ваш лендінг **не** використовує Tailwind, можна:
  - або підключити Tailwind;
  - або замінити класи на власні (HTML‑структура максимально проста).

## Мовна підтримка

Кожен showcase‑компонент має **перемикач мови** всередині (uk/en).
Якщо треба керувати мовою ззовні (з глобального i18n), можна:
- передати `defaultLanguage` як проп (простий рефакторинг),
- або вирізати внутрішній toggle й замінити `useState` на значення з контексту.

## Безпека та AI

- У прикладах згадуються:
  - **захищені відеодзвінки** (аналог поточного Jitsi‑інтерфейсу),
  - **інструменти на дзвінку** (нотатки, чек‑листи, таймер, аналітика),
  - **AI‑асистент / супервізор**, який пропонує підказки та фокуси для сесії.
- Це **візуальні / демонстраційні блоки**:
  - немає реальних дзвінків;
  - немає реальних AI‑запитів;
  - немає справжніх клієнтських даних.

## Для редагування текстів

- Можна **вільно змінювати тексти** всередині об’єктів перекладу (`copy.uk`, `copy.en`).
- Не змінювати:
  - назви експортованих компонентів (`StudentsExamplesShowcase`, `ClinicsExamplesShowcase` тощо),
  - сигнатуру спільної картки `LandingExampleCard` (якщо її використовуєте в кількох місцях).
